📘 API — Carga de Planilla de Partido

Permite registrar el resultado de un partido junto con las estadísticas de jugadores de ambos equipos.

🔗 Endpoint

POST /api/cargaPlanillaPartido

✅ Validaciones

🎯 Datos del partido

CampoTipoRequeridoDescripción
partido_idinteger✅ID del partido existente
res_localinteger ≥ 0✅Goles equipo local
res_local_pinteger ≥ 0⚠️ nullablePenales local
res_visitanteinteger ≥ 0✅Goles visitante
res_visitante_pinteger ≥ 0⚠️ nullablePenales visitante

🧑‍⚖️ Árbitros

CampoTipoRequerido
arbitro1string max 255✅
arbitro2string max 255⚠️ nullable
arbitro3string max 255⚠️ nullable
informearbitrostring⚠️ nullable

🏠 Equipo Local

equipo_local debe ser un array de jugadores.

CampoTipoRequeridoValidación
jleidinteger✅Debe pertenecer al equipo local
camisetainteger 0-9999⚠️ nullable
gfinteger ≥ 0✅
gcinteger ≥ 0✅
amarilla0 o 1✅
dobleamarilla0 o 1✅
roja0 o 1✅
jugo0 o 1✅

🚗 Equipo Visitante

equipo_visitante debe ser un array de jugadores.

CampoTipoRequeridoValidación
jleidinteger✅Debe pertenecer al equipo visitante
camisetainteger 1-9999⚠️ nullable
gfinteger ≥ 0✅
gcinteger ≥ 0✅
amarilla0 o 1✅
dobleamarilla0 o 1✅
roja0 o 1✅
jugo0 o 1✅

❌ Error de validación

Notas:
  • partido_id debe existir en la tabla partidos.
  • Cada jugador debe pertenecer al equipo correcto.
  • Los flags disciplinarios son binarios (0 o 1).
  • Todos los campos deben existir en el json.
